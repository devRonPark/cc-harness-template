# grill-me — 인터뷰 기반 PRD 작성

아이디어를 집요하게 인터뷰해서 `docs/PRD.md` 초안까지 만드는 것이 이 스킬의 책임이다.
인터뷰만 하고 끝내지 않는다 — **PRD 파일 작성까지가 완료 조건.**

## 시작 전

1. `docs/PRD.md`가 이미 있으면 읽는다. 처음부터 다시 묻지 말고 **빈 곳(Open Questions, 미기재 섹션)만** 인터뷰한다.
2. 코드베이스로 답할 수 있는 질문은 사용자에게 묻지 말고 직접 탐색해서 답한다.

## 인터뷰 규칙

- **질문은 반드시 한 번에 하나.** 여러 질문 동시 투척 금지.
- **모든 질문에 권장 답을 함께 제시한다.** "어떻게 할까요?"가 아니라 "A를 권장합니다. 이유: X. 동의?"
- 가능하면 객관식으로. 답이 갈리는 지점(트레이드오프)만 묻고, 뻔한 건 기본값으로 깔고 넘어간다.
- 설계 트리를 가지별로 내려간다: 목적 → 사용자 → 핵심 기능 → 비기능 요구 → 성공 기준. 앞 답이 뒤 질문을 결정한다.
- 사용자가 모호하게 답하면 구체적 시나리오로 되물어 확정한다.

## 종료 조건

다음이 채워지면 인터뷰를 멈추고 PRD를 쓴다:

- 목적과 대상 사용자가 한 문장으로 나옴
- 핵심 기능 3±2개와 명시적 Non-goals
- 측정 가능한 성공 기준 최소 1개

완벽을 기다리지 않는다. 답이 안 나온 항목은 **Open Questions**에 남기고 초안을 낸다.

## 산출

1. `docs/templates/PRD.md` 골격을 복사해 `docs/PRD.md` 작성. 인터뷰 중 확정된 결정은 Decisions 섹션에 근거와 함께 기록.
2. 작성 후 사용자에게 검토 요청. 승인되면 후속 안내:
   - User Flow·Architecture 보완 → `docs/templates/UserFlow.md`, `docs/templates/Architecture.md` 골격 사용
   - 문서 확정 후 `/harness-plan`으로 Plans.md Task 생성

## 중단·재개

인터뷰가 중간에 끊기면 그때까지 확정된 내용으로 `docs/PRD.md`를 저장하고
미해결 항목을 Open Questions에 남긴다. 재실행 시 그 지점부터 이어간다.
