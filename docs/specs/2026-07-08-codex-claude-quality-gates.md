# Claude/Codex Quality Gates Plan

Date: 2026-07-08

## Summary

Claude Code can receive ponytail and caveman behavior through plugins and hooks.
Codex does not run those Claude Code plugins automatically. This template keeps
the Claude plugin surface, but moves the durable engineering principles into a
repository document that both Claude and Codex workflows can read directly:
`agents/quality-gates.md`.

## Current Verification

- `AGENTS.md` exists as the Codex entrypoint and maps Claude slash workflows to
  `.agents/skills/*`.
- `.agents/skills/harness-work/SKILL.md` and
  `.agents/skills/harness-review/SKILL.md` exist and can reference shared gates.
- `init.sh` copies `agents/` and `.agents/skills/`, so new projects inherit both
  the shared gate document and Codex skill procedures.
- `tasks/index.json` is the task source of truth. `Plans.md` is generated by
  `scripts/sync_plans.py`.

## Decision

Add `agents/quality-gates.md` as the common quality gate for Claude and Codex.
It captures:

- ponytail-like engineering checks: YAGNI, reuse before new code, scope control,
  over-abstraction rejection, and task split triggers.
- caveman-like reporting checks: short status, findings first, verification
  evidence first, and no unnecessary background.

Claude Code keeps ponytail/caveman as Claude-only plugin enhancements. Codex
uses the same principles by reading `AGENTS.md` and `.agents/skills/*`, which now
point at `agents/quality-gates.md`. No fake `$ponytail` or `$caveman` Codex skill
is added because that would imply automatic plugin behavior that Codex does not
provide.

## Alternatives Rejected

- Create Codex `$ponytail` and `$caveman` skills: rejected because they would be
  manual commands, not automatic hooks, and would make user expectations drift.
- Remove Claude plugin documentation: rejected because Claude Code users still
  benefit from the plugin layer and setup instructions.
- Put the rules only in `AGENTS.md`: rejected because Claude and Codex skills
  need a stable, role-neutral reference that also fits the existing `agents/`
  procedure directory.

## Files To Change

- `agents/quality-gates.md`: new common quality gate.
- `.agents/skills/harness-work/SKILL.md`: implementation gate reference.
- `.agents/skills/harness-review/SKILL.md`: review gate reference.
- `.agents/skills/harness-plan/SKILL.md`: planning split reference.
- `AGENTS.md`, `CLAUDE.md`, `README.md`, `BLUEPRINT.md`: document Claude-only
  plugin enhancement vs Codex built-in procedure.
- `.harness/CONTEXT_INDEX.md` and `templates/skeleton/.harness/CONTEXT_INDEX.md`:
  file role index updates.
- `tasks/index.json` and generated `Plans.md`: task tracking.
